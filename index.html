CLOUD COMPUTING


BEFORE CLOUD COMPUTING


    Before cloud computing, the following steps were followed in deploying an application:


  1. Create a website.

  2. Deploy it on a physical machine(server) that had an operating system and set up the configurations and middleware needed to run it.

  3. Set up the runtime to run the application, e.g. Apache, Tomcat or IBM Websphere Liberty app server(Java).

  4. Link your app to a database.

    In order to expose your app to your client, one needed all the following:


  1. Set up an IP address and domain name.

  2. Network configurations.

  3. Physical location for your servers.

  4. Electricity for cooling and running your servers.

  5. Security.

  6. A team of experts to install, configure, test, run, secure and update these resources to keep your website running.

    Challenges faced before cloud computing

  1. Cost - It was costly to build reliable and maintainable software. You likely had to build your own infrastructure that might have been needed to be turned into a data centre which included servers, network equipment, data storage, etc. Then, a team of experts needed to be hired to handle all those resources. All these factors made it difficult for small and mid-size businesses to develop their own software and keep updating it.

  2. Scalability - A high demand for your application by clients meant that you needed to scale up its capacity, which required more resources and some downtime to integrate and upgrade those resources.

  3. Reliability - One required downtime to perform server operations such as maintenance and updates. Some issues that might have caused downtime are power outages, hardware problems, general network issues etc ,which promoted unreliability.

  4. Security - Necessary for all levels: application, network, infrastructure,and physical resources.

  5. Mobility - Part of the team worked onsite to set up the infrastructure and configure the network.


CLOUD COMPUTING OVERVIEW


Enables ubiquitous computing where computing is available anytime and anywhere using any device, in any location and in any format.

The delivery of on-demand computing services ranging from applications to servers, databases, software, analytics and intelligence over the internet, i.e. the cloud, on a pay-as-you-go basis.

    Characteristics of the Cloud

    The cloud makes hardware resources readily available and quick to configure, which shortens the time needed by developers to show a working version of their product. It also allows for reuse of the same resources for successive projects, which is more cost-efficient.


  1. On-demand resources - Have it when you need it with no need for tiresome preparation, downloads and installations.

  2. Self-service - A customer can provision resources themselves by accessing a self-service portal and requesting the resources they want.

  3. Ubiquitous access - Access the cloud from anywhere through internet connection and a cloud account.

  4. Resource pooling - Pooling hardware resources and abstracting them so that when they're not being used by one customer, they can be used by the next, instead of sitting idle.

  5. Rapid elasticity - Scaling up or down resource consumption is available on demand with any quantity and at any time.

  6. Measured service - Pay only for what you use, which helps monitor resource wastage.

    Benefits of cloud computing

  1. Cost - Eliminates the capital expense of buying hardware, software and setting up on-site data centers, server racks, IT infrastructure managers etc.

  2. Speed - Most cloud computing services are provided on demand, so vast computing resources can be supplied in minutes, giving businesses flexibility and taking pressure off the planning capacity.

  3. Global Scale - Has the ability to scale elastically i.e. delivering the correct amount of IT resources e.g. more or less computing power, storage, band-width, when neeeded and from the right geographical location.

  4. Productivity - Enhances productivity of IT teams, as they can focus on achieving more important goals as opposed to setting up hardware at on-site data centers, software patching and other time-consuming IT management tasks.

  5. Reliability - Makes data back-up, disaster recovery and business continuity easier and less expensive since data can be mirrored at multiple redundant sites on the cloud provider's network.

  6. Security - Helps protect one's data, apps and infrastructure from potential threats by providing policies, technologies and controls that strengthen security posture.

    Factors that have led to the growth of cloud computing

  1. Applications with a short lead time to delivery - Today's apps require a short time to delivery.The need for developers to get their product to the market in the shortest time possible and quickly get feedback, iterate on the idea to make the product better and faster has made the cloud's hardware resources readily available and quick to configure, therefore shortening the time needed by devs to show a working version of their product.

  2. Developers expect to have programming language options and interact with predefined services - Cloud computing provides pre-packaged language support as opposed to the traditional DIY environment.

  3. Modern applications must be able to scale and be managed dynamically - Devs want to be able to add more resources to a specific app (vertical scaling) or add add duplicate instances of the app (horizontal scaling) to handle increased customer load. Cloud platforms provide standardized methods to scale applications.

  4. Developers expect the pay-as-you-go utility computing billing method - This means that they pay for only their cloud usage.

    Types of Cloud Computing

  1. Public Cloud - Owned and operated by a 3rd party cloud service provider, e.g. Microsoft Azure. All hardware, software and supporting infrastructure is owned and managed by the cloud provider.

  2. Private Cloud - Services and infrastructure are maintained on a private network. Resources are exclusively used by the business. Can be physically located on the company's on-site data center.

  3. Hybrid Cloud - A combination of public and private cloud, bound together by tech that allows data and apps to be shared between them, which in turn gives flexibility, more deployment options and helps optimise existing infrastructure, security and compliance.

    Types of Cloud Services/Models

    To build an app, a cloud provider must provide the infrastructure, platform, OS, networking components etc. Using the pizza analogy to explain IaaS, PaaS and SaaS:

  1. IaaS (Infrastructure as a Service) - You order hardware and an infrastructure often managed by you. You deploy only the middleware, runtime and your app. The infrastructure is like pre-made pizza and you bake it to your liking. Therefore, a cloud provider offers clients pay-as-you-go access to virtual machines, servers storage, networks, operating systems and other computing resources in the cloud. Helps a user avoid the expense and complexity of buying and managing their own data center infrastructure.

  2. Paas (Platform as a Service) - More like a pizza getting delivered. The pizza is ready to be eaten and only needs drinks to go with it. The cloud provider offers access to the platform and runtime and you only need to push the application. Therefore, a cloud provider offers access to a cloud-based environment in which the user can build and deliver apps,i.e. supplies and manages the underlying infrastructure.

  3. Saas (Software as a Service) - Using an application hosted at the cloud provider. Similar to going to a restaurant and enjoying your pizza there. Therefore, all underlying infrastructure, middleware, app software and data are located in the service provider's data center. A cloud provider delivers software and applications that are ready to be consumed via the internet, e.g. email, calendar, office tools.

  4. Serverless Computing - The cloud provider automatically supplies, scales and manages infrastructure required to run the code, therefore making it easier for a developer to build applications faster. Tip: The term 'serverless' comes from the fact that the management and infrastructure provision tasks are invisible to the developer.

    Uses of Cloud Computing

  1. Create cloud-native apps - Allows for quick building, deployment and scaling of web, mobile applications and API.

  2. Store, back-up and retrieve data - Protects your data by transferring it over the internet to an offsite cloud storage system that's accessible from any location and device.

  3. Stream audio and video - Connect with an audience anywhere at ant time on any device through high-def audio and video with global distribution.

  4. Deliver software on demand(Saas) - Lets a user offer the latest software versions and updates to customers, any time they need it and anywhere they are.

  5. Test and build applications - The use of cloud infrastructure that can be easily scaled up or down reduces app development cost and time.

  6. Analyse data - Joins data from across locations and teams in the cloud then uses machine learning and artificial intelligence to get more insights on the data.

  7. Embed intelligence - Use intelligent models that help give valuable insights on the data captured, thereby keeping the audience engaged.

    Cloud-native Applications

    Capitalize on the scalability and flexibility of the cloud.

  1. Apps are broken into separate services a.k.a microservices.

  2. microservices can be developed in different programming languages (polygot development).

  3. microservices communicate with each other using an agreed-upon protocol, e.g. REST or gRPC.

  4. Microservices work together to make up an app, yet each can be independently scaled, continuously improved and quickly iterated via automation and orchestration processes.

    Cloud-native vs Cloud-enabled

    Cloud-enabled apps are those that were developed for deployment in a traditional data centre but were later changed to run in cloud environment.

DEPLOYING FIRST CLOUD APPLICATION WITH IBM CLOUD CLI

Deploy and configure a Node.js Cloud Foundry app to IBM Cloud.

    Tools needed/install

  1. IBM Cloud CLI (Command Line Interface) - A command line interface used to build and deploy IBM cloud applications.

  2. Git - a distributed version-control system for tracking source code during software development.

  3. Node.js - an open-source JavaScript runtime environment used to execute JavaScript code outside a web browser.

  4. npm - The default package manager for Node.js.

    5. Tip: Connnect to the IBM cloudant NoSQL database on your dashboard













    npm Commands

  1. npm install - Installs node.js modules.

  2. npm audit fix - Fixes any vulnerabilities to modules installed.

  3. node v - Checks version of node installed.

  4. npm v - Checks version of npm installed.

  5. npm start - Runs your node.js application locally.

    IBM Cloud CLI Commands

  1. ibmcloud login - Logs you in to IBM Cloud.

  2. ibmcloud --version - Checks the version of IBM Cloud CLI installed.

  3. ibmcloud target --cf - Sets up your IBM Cloud Foundry API endpoint.

  4. ibmcloud cf push - Pushes your app to IBM Cloud.

  5. ibmcloud cf apps - Checks status of the pushed app

  6. ibmcloud plugin install container-registry - Installs IBM Cloud container registry plugin.

  7. ibmcloud plugin show container-registry - Shows container registry plugin installation details.

  8. ibmcloud plugin install kubernetes-service - Installs IBM Cloud kubernetes service plugin.

  9. ibmcloud plugin show container-service - Shows kubernetes service plugin installation details.

  10. ibmcloud plugin install dev - Installs developer tools plugin.

  11. ibmcloud plufin show dev - Shows developer tools plugin installation details.

  12. ibmcloud cf help -a - Lists out the available IBM Cloud commands.

  13. ibmcloud cf ssh app_name - Explores how your app's directories are structured after deployment.
